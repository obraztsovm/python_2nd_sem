Тема: NumPy, векторизация, бродкастинг и линейная алгебра.
Данные: olist_order_items_dataset.csv, olist_products_dataset.csv.
Задача:
Вам поручено оптимизировать ядро системы расчетов в бразильском маркетплейсе Olist. Поскольку транзакций миллионы, использование стандартных циклов Python недопустимо по соображениям производительности.


Задание 1: Битва производительности (Loops vs NumPy)
Необходимо рассчитать итоговую стоимость позиции в заказе с учетом налога и динамической скидки. Используйте файл olist_order_items_dataset.csv.
Логика: Итоговая цена = (price * (1 - discount)) + freight_value.
Скидка: Если price > 500 BRL — 15%, если > 100 BRL — 10%, иначе 0%.
Реализация А: Напишите функцию на чистом Python, которая итерируется по спискам и применяет условия.
Реализация Б: Используйте np.select или булевы маски NumPy для векторизованного расчета.
Профилирование: Сравните скорость с помощью %timeit. Ожидаемое ускорение — более чем в 50 раз.


Задание 2: Интеллектуальный фильтр и Fancy Indexing
Используйте файл olist_order_items_dataset.csv для поиска аномалий.
1.
Булево индексирование: Выделите все записи, где цена ( price) выше среднего значения по датасету более чем на 3 стандартных отклонения (3σ).
2.
Fancy Indexing: Создайте массив индексов для 100 случайных заказов и выберите соответствующие им строки из таблицы, используя только средства NumPy.


Задание 3: Бродкастинг (Объемный вес и логистика)
Используйте файл olist_products_dataset.csv. Для расчета стоимости хранения нам нужен объемный вес товаров.
1.
Векторизация: Рассчитайте объемный вес по формуле: Vweight=(L×H×W)/6000.
Используйте колонки product_length_cm, product_height_cm, product_width_cm.
2.
Broadcasting: Представьте, что у вас есть матрица цен товаров (1000 товаров × 5 складов). Умножьте всю матрицу на вектор региональных коэффициентов (размера 5) за одну операцию без использования циклов.


Задание 4: Векторное сходство (Линейная алгебра)
Вам нужно реализовать простейший движок рекомендаций на основе характеристик товаров из olist_products_dataset.csv.
1.
Подготовка: Создайте матрицу признаков (например: вес, длина, высота, ширина) для первой тысячи товаров. Нормализуйте данные.
2.
Матричные операции: Реализуйте расчет косинусного сходства (Cosine Similarity) между вектором конкретного «выбранного» товара и всей матрицей признаков.
3.Инструментарий: Используйте только матричное умножение ( @ или np.dot) и np.linalg.norm. Никаких внешних библиотек вроде scikit-learn.


Требования к сдаче
Zero Loops: В расчетных ячейках полностью отсутствуют циклы for и while.
Profiling: В ноутбуке присутствуют результаты выполнения %timeit для Задания 1.
Vector Thinking: Все операции в Задании 4 выполнены в матричном виде, а не через итерацию по векторам.
